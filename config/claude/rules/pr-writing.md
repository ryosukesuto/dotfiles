# PR / コミットメッセージは共有コンテキストで書く

PR title / body / commit message には、チーム外から参照できない個人固有の語彙を書かない。

## 避ける表現

- 個人 skill 名（例: `/harness-audit`, `/post-merge`, `/work-log` など、チームに共有されていない自作スキル）
- 個人 dotfiles / ローカルノートのパス（例: `~/dotfiles-private/...`、Obsidian デイリーノートの相対リンク）
- チームのリポジトリに存在しないローカル運用スクリプト名

## 理由

レビュワーにはこれらのリソースが共有されておらず参照不能。PR の背景が追えず、レビュー効率が落ちる。

## 書き換え方

作業着手のトリガーが個人スキル起動でも、PR 本文にはその事実を書かない。言及している全リソースが org / チームから参照可能かを確認し、参照不能なら以下のいずれかに翻訳する。

- org リポに既に存在する先行事例（`<org>/<repo>` の該当ファイル）
- Kibela / チーム Wiki エントリ
- リポジトリ内の ADR（`docs/adr/`）
- Linear Issue / Slack permalink
