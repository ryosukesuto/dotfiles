# 人に読ませる文章は共有前提から設計する

PR本文、レビューコメント、Slack投稿、議事録、ADR、設計書、解説文など、人に読ませる文章を作るときに適用する。

## 書く前に決める

1. 読み手を決める — 誰が読み、どの情報やリンクへアクセスでき、何を既に知っているか
2. 理解の更新を決める — 読後に何を理解・判断・実行できれば成功かを1文で表す
3. 情報の状態を分ける — 事実、観測、推測、提案、決定、未確認を区別する

読み手は文書種別と共有先から推定する。想定によって内容が大きく変わる場合だけユーザーへ確認する。

## 書いている間の基準

- 各段落・箇条書きは、状況の更新、必要な因果の説明、判断材料、次の行動のいずれかを担う
- 判断に必要な因果関係は書き手が説明する。根拠のない抽象的な因果だけを削る
- 読み手から参照できない個人用語やローカル文脈は、共有可能な事実へ翻訳する
- 全員を初学者と仮定しない。読み手が理解の更新に必要とする差分だけを補う
- 内容のないセクションを一般論や`N/A`で埋めない。不要なセクションは省く
- 文書の構造を案内するだけで状況を更新しない文は削る

## 出す前に確認する

- 読み手が中心的な主張や変更を1文で言い直せる
- 決定済み、提案中、未確認を区別できる
- 重要な主張の根拠を辿れる
- 必要な次の行動、または行動不要であることが分かる
