# クイックリファレンス

Claude Codeを効果的に使うためのユーザー向けガイダンス。

## プロンプトのベストプラクティス

### 検証方法を与える
テストケースや期待出力を含めると、Claudeが自己検証できる。

```
悪い例: 「パーサーを作って」
良い例: 「パーサーを作って。入力 "a,b,c" → 出力 ["a","b","c"]、空文字 "" → []」
```

### 具体的に指示する
曖昧な指示より、制約や条件を明示する。

```
悪い例: 「テスト追加して」
良い例: 「エッジケース含むテスト追加。mocking不要、実際のファイルI/O使用」
```

## CLAUDE.mdの書き方

### 簡潔に保つ
- Claudeが推測できない情報だけ記載
- プロジェクト固有のルール、ディレクトリ構成、コマンド
- 一般的なベストプラクティスは不要（Claudeは既に知っている）

### 避けるべき内容
- 「コードは読みやすく書く」のような当たり前のこと
- 長い説明文（箇条書きで簡潔に）
- 重複した情報
