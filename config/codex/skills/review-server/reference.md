# review-server reference

直接呼び出し時の出力フォーマット。Codex版 `review-pr` のspecialist promptからJSON形式を指定された場合は、親promptの形式を優先する。

## 出力フォーマット

```markdown
## レビュー結果

変更内容: X files (+XXX/-XXX lines) | CI: Pass/Fail/未確認
総合評価: X/10点

### 概要
PRの目的と変更内容を1-2文で要約。

### レビューステータス
- [ ] 変更要求（P0項目の修正後に再レビュー）
- [ ] 承認保留（質問への回答待ち）
- [ ] 承認（マージ可能）

---

## P0: マージ前に必須
### [file:line] 問題の要約
問題: 何が問題か
修正案: 具体的な修正方法

## P1: 強く修正推奨
### [file:line] 問題の要約
問題: 何が問題か
修正案: 改善方法

## P2: 対応推奨
- [file:line] 簡潔な提案

## P3: 任意改善
- [file:line] 簡潔な提案

---

## クロスリポジトリ影響（該当時のみ）
- server-config: {影響内容}
- ops: {影響内容}

## 既存コメントへの補足（該当時のみ）
- [@reviewer file:line] 同意/異議の理由

## 良かった点（任意）
- [file:line] 良い実装の理由

## 注意事項（任意）
- Breaking Changes: なし/あり
- ロールバック: 容易/注意が必要
- 自動生成ファイルの更新: 必要/不要
```

## Diff-Only Review

大規模PRで先入観なしの観点を追加するときは、PR説明を閉じてdiffだけを読む。

確認観点:

- 変更の意図がdiffから読み取れるか
- テストで検証されていないエッジケースがあるか
- 暗黙の前提に依存している箇所があるか
- クリーンアーキテクチャの依存方向違反があるか

この結果は通常レビューに統合し、重複指摘はまとめる。
